## 🎯 现在你只需要做这 2 件事：

### **第 1 步：部署后端（5 分钟）**

#### 👉 点击这里：**https://railway.app**

1. **Sign in with GitHub** （用 GitHub 登录）
2. **New Project** → **Deploy from GitHub repo**
3. 选择 `bilibili-analyzer`
4. 等待自动构建（约 3 分钟）
5. 构建完成后，点击项目 → **Variables**
6. 添加两个环境变量：
   ```
   CORS_ORIGIN=*
   BILIBILI_COOKIE=你的B站Cookie
   ```

#### 🔑 Cookie 怎么获取？

```
1. 浏览器打开 https://www.bilibili.com 并登录
2. 按 F12 打开开发者工具
3. Network 标签 → 刷新页面（F5）
4. 点击左边任意请求
5. 右边找到 "Cookie" 字段，复制整行
6. 粘贴到 Railway 的 BILIBILI_COOKIE 变量中
```

7. Railway 会自动重启服务
8. 点击 **Settings** → 复制域名（类似 `xxx-production.up.railway.app`）

✅ **后端完成！记住这个域名！**

---

### **第 2 步：部署前端（3 分钟）**

#### 👉 点击这里：**https://vercel.com**

1. **Continue with GitHub** （用 GitHub 登录）
2. **Add New Project** → **Import Git Repository**
3. 选择 `bilibili-analyzer`
4. **Configure Project**：
   - Framework Preset: **Vite**
   - Root Directory: **frontend**
5. **Environment Variables**（重要！）：
   ```
   Key: VITE_API_URL
   Value: https://你的后端地址.railway.app
   ```
   （填入刚才 Railway 的域名）

6. 点击 **Deploy**
7. 等待 1-2 分钟

✅ **前端完成！**

## 🆘 遇到问题？

### 常见问题速查：

**Q: Railway 构建失败？**
- 检查日志（Logs 标签）
- 确认 `railway.json` 存在

**Q: Vercel 构建失败？**
- 确认 Root Directory 是 `frontend`
- 查看 Vercel 的 Deploy Logs

**Q: 提示 CORS 错误？**
- Railway 添加环境变量：`CORS_ORIGIN=*`

**Q: API 限流？**
- 必须配置 `BILIBILI_COOKIE`
