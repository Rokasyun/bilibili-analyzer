## 📋 接下来你需要做的 3 个步骤

### **Step 1️⃣: 创建 GitHub 仓库并推送**

#### 1.1 在 GitHub 上创建新仓库

打开浏览器访问：https://github.com/new

**填写信息：**
- Repository name: `bilibili-analyzer`（或其他你喜欢的名字）
- 选择：**Public** 或 **Private**（推荐 Public）
- ❌ 不要勾选 "Add a README file"
- ❌ 不要勾选 ".gitignore"
- ❌ 不要选择 License

点击 **"Create repository"**

#### 1.2 复制 GitHub 提供的推送命令

创建成功后，GitHub 会显示一段命令，类似：

```bash
git remote add origin https://github.com/你的用户名/bilibili-analyzer.git
git branch -M main
git push -u origin main
```

**或者直接在 PowerShell 中执行以下命令（替换为你的仓库地址）：**

```bash
# 请将 YOUR_USERNAME 替换为你的 GitHub 用户名
git remote add origin https://github.com/YOUR_USERNAME/bilibili-analyzer.git
git branch -M main
git push -u origin main
```

**如果遇到错误：**
- 如果提示远程仓库已存在，说明你已经创建过同名仓库
- 使用 `git remote -v` 查看当前远程地址
- 使用 `git remote set-url origin 新地址` 修改

---

### **Step 2️⃣: 部署后端到 Railway**

#### 2.1 访问 Railway

打开：https://railway.app

**登录方式：**
- 点击 "Start a New Project"
- 选择 "Login with GitHub"
- 授权 Railway 访问你的 GitHub 仓库

#### 2.2 创建新项目

1. 点击 **"New Project"**
2. 选择 **"Deploy from GitHub repo"**
3. 找到并选择 `bilibili-analyzer` 仓库
4. Railway 会自动识别并加载配置

#### 2.3 等待自动构建

Railway 会自动：
- 读取 `railway.json` 配置
- 安装依赖
- 构建项目
- 启动服务

**预计耗时：** 2-5 分钟

#### 2.4 配置环境变量

构建完成后（或构建过程中）：

1. 点击项目卡片进入详情页
2. 切换到 **"Variables"** 标签
3. 添加以下环境变量：

```
CORS_ORIGIN=https://your-app.vercel.app
BILIBILI_COOKIE=你的 B 站 Cookie（可选但强烈推荐）
```

**如何获取 B站 Cookie：**
1. 浏览器访问 https://www.bilibili.com 并登录
2. 按 F12 打开开发者工具
3. 切换到 "Network" 标签
4. 刷新页面（F5）
5. 点击左侧任意请求
6. 在右侧找到 "Request Headers"
7. 复制 "Cookie" 字段的完整内容
8. 粘贴到 Railway 的变量值中

#### 2.5 获取后端地址

 Railway 会分配一个公网地址，格式类似：
```
https://bilibili-analyzer-production.up.railway.app
```

**复制这个地址，后面要用！**

**如何找到它：**
1. 在 Railway 项目页面
2. 点击 **"Settings"** 标签
3. 找到 **"Domains"** 部分
4. 复制分配的域名

---

### **Step 3️⃣: 部署前端到 Vercel**

#### 3.1 访问 Vercel

打开：https://vercel.com

**登录方式：**
- 点击 "Continue with GitHub"
- 授权 Vercel 访问你的 GitHub 仓库

#### 3.2 Import 项目

1. 点击 **"Add New Project"**
2. 选择 **"Import Git Repository"**
3. 找到 `bilibili-analyzer` 仓库
4. 点击 **"Import"**

#### 3.3 配置项目

**Framework Preset:** 
- 选择 **"Vite"**

**Root Directory:** 
- 点击 "Edit"
- 输入 `frontend`
- 这告诉 Vercel 前端代码在 frontend 文件夹

**Build Command:** 
- 保持默认：`npm run build`

**Output Directory:** 
- 保持默认：`dist`

#### 3.4 配置环境变量

在部署前，需要添加环境变量：

1. 点击 **"Environment Variables"**
2. 点击 **"Add Variable"**
3. 添加以下变量：

```
Key: VITE_API_URL
Value: https://你的后端地址.railway.app
```

**注意：** Value 要替换成你在 Railway 获得的后端地址！

#### 3.5 开始部署

1. 确认所有配置正确
2. 点击 **"Deploy"**
3. 等待 Vercel 构建（约 1-3 分钟）

#### 3.6 获取前端地址

部署成功后，Vercel 会分配一个地址，格式类似：
```
https://bilibili-analyzer.vercel.app
```

**恭喜！你的应用已经上线了！🎉**

## ⚠️ 常见问题解决

### 问题 1: Railway 构建失败

**可能原因：**
- Node.js 版本不兼容
- 依赖安装失败

**解决方法：**
1. 检查 Railway 的构建日志
2. 确认 `railway.json` 配置正确
3. 尝试在本地运行 `npm run build:backend` 看是否有错误

### 问题 2: Vercel 构建失败

**可能原因：**
- Root Directory 设置错误
- 依赖问题

**解决方法：**
1. 确认 Root Directory 设置为 `frontend`
2. 检查 Vercel 的构建日志
3. 在本地运行 `cd frontend && npm run build` 测试

### 问题 3: CORS 错误（跨域）

**症状：**
- 前端无法调用后端 API
- 浏览器控制台显示 CORS 错误

**解决方法：**
1. 在 Railway 添加环境变量 `CORS_ORIGIN`
2. 值设置为你的 Vercel 域名（不带斜杠）
   ```
   CORS_ORIGIN=https://bilibili-analyzer.vercel.app
   ```
3. Railway 会自动重启

### 问题 4: API 限流

**症状：**
- 提示「请求被限流」

**解决方法：**
1. 在 Railway 配置 `BILIBILI_COOKIE` 环境变量
2. 按照上面的方法获取 Cookie 并填入
